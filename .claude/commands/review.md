# コードレビュー

`git diff HEAD` で変更内容を取得し、以下の観点でレビューしてください。

## レビュー観点

### 1. バグ・ロジックの誤り

- 条件分岐の漏れ、境界値エラー、null/undefined の未処理
- 非同期処理の誤り（await 忘れ、競合状態）

### 2. セキュリティ

- XSS、SQL インジェクション、パストラバーサルなどの脆弱性
- 機密情報のハードコーディング

### 3. 型安全性（TypeScript）

- `any` の使用、型アサーションの乱用
- 不完全な型定義

### 4. パフォーマンス

- 不要な再レンダリング、N+1 問題、重い処理のホットパスへの追加

### 5. コード品質

- 重複コード、不要なコメント、命名の不一致
- 既存コンポーネント・ユーティリティで代替できる箇所

## 出力形式

変更ファイルごとに以下の形式でまとめてください：

```
### ファイル名
- 🔴 重大: ...
- 🟡 軽微: ...
- 🟢 提案: ...
```

問題がなければ「✅ 問題なし」と記載してください。
